# 灵光命盘本地操作说明

## 1. 填写 DeepSeek API Key

打开项目根目录里的 `.env` 文件：

```text
/Users/zisazhou/Documents/灵光/.env
```

把这一行：

```env
DEEPSEEK_API_KEY=把你的真实API_KEY粘到这里
```

改成：

```env
DEEPSEEK_API_KEY=你的真实API_KEY
```

不要把 API Key 写进 `script.js`，也不要发到聊天里。

## 2. 启动网站

在项目目录运行：

```bash
npm start
```

然后打开：

```text
http://127.0.0.1:4177
```

不要用 `file:///Users/.../index.html` 打开，因为那样前端无法请求后端接口。

## 3. 测试排盘

页面里输入：

```text
出生年、出生月、出生日、出生时间、性别
```

点击：

```text
生成八字四柱
```

此时会调用：

```text
POST /api/paipan
```

如果成功，页面会显示年柱、月柱、日柱、时柱。

当前排盘来源已经改为：

```text
lunar-javascript
```

也就是说，四柱和大运起运由本地确定性排盘库生成，不再由 DeepSeek 生成。DeepSeek 只负责后面的付费分析报告。

大运起运规则由 `.env` 里的这个字段控制：

```env
BAZI_YUN_SECT=1
```

如果你发现和参考网站起运年龄差几天或几个月，可以改成：

```env
BAZI_YUN_SECT=2
```

改完后需要重启服务。

## 4. 测试付费报告

先生成八字四柱，再点击：

```text
解锁全部命理 ¥9.9
```

此时会调用：

```text
POST /api/report
```

现在还没有接真实支付，只是模拟解锁，用来测试报告质量。

报告生成成功后，后端会保存到：

```text
/Users/zisazhou/Documents/灵光/data/reports.json
```

如果 `.env` 配置了 `DATABASE_URL`，则会保存到 PostgreSQL 的 `reports` 表，本地 JSON 只作为兜底。

页面会返回：

```text
查询码
查询链接
```

查询链接格式：

```text
http://127.0.0.1:4177/report/查询码
```

用户刷新页面后，可以通过这个链接重新查看报告。

## 5. 常见问题

### 127.0.0.1 拒绝连接

说明后端服务没启动。运行：

```bash
npm start
```

### 缺少 DEEPSEEK_API_KEY

说明 `.env` 里还没填真实 API Key，或者服务没有重启。

填完 `.env` 后，需要停止服务再重新运行：

```bash
npm start
```

### Insufficient Balance

DeepSeek 返回余额不足。说明 API Key 已经被后端识别，也成功请求到了 DeepSeek，但账户没有可用额度。

处理方式：

```text
去 DeepSeek 控制台充值 / 开通额度
```

充值后不用改代码，重新点击页面里的“生成八字四柱”即可。如果服务已经停了，再运行：

```bash
npm start
```

### 排盘结果和参考网站不一致

先不要上线。记录这个生日、时间、性别，以及参考网站的正确四柱。

现在 `/api/paipan` 已经接入 `lunar-javascript`。如果仍然不一致，优先确认差异是不是来自：

```text
1. 子时规则不同
2. 是否使用真太阳时
3. 年柱是否按立春
4. 月柱是否按节气
5. 大运起运折算规则不同
6. 参考网站自身采用的流派不同
```

如果你决定以某个网站为唯一标准，就需要按那个网站的规则继续适配。
