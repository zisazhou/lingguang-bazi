# 网页端部署说明

## 推荐路线

第一版建议部署到 Render 的 Node Web Service。

不要先做微信小程序。当前项目有 Node 后端、DeepSeek API Key、报告保存逻辑，直接上纯静态平台不合适。

## 部署前确认

项目根目录应包含：

```text
server.js
package.json
render.yaml
index.html
styles.css
script.js
```

本地先确认能跑：

```bash
npm start
```

打开：

```text
http://127.0.0.1:4177
```

## Render 部署步骤

1. 把项目推到 GitHub。

2. 打开 Render，创建 Web Service。

3. 连接这个 GitHub 仓库。

4. 构建命令：

```bash
npm install
```

5. 启动命令：

```bash
npm start
```

6. 环境变量：

```env
HOST=0.0.0.0
DEEPSEEK_API_KEY=你的真实API_KEY
DEEPSEEK_API_URL=https://api.deepseek.com/chat/completions
DEEPSEEK_MODEL=deepseek-chat
BAZI_YUN_SECT=1
DATABASE_URL=你的PostgreSQL连接字符串
DATABASE_SSL=true
```

不要把 `DEEPSEEK_API_KEY` 和 `DATABASE_URL` 提交到 GitHub。

## 数据库

代码现在支持 PostgreSQL。

如果配置了：

```env
DATABASE_URL=...
```

报告会保存到 PostgreSQL 的 `reports` 表。

如果没有配置 `DATABASE_URL`，才会退回本地 JSON 文件：

```text
/Users/zisazhou/Documents/灵光/data/reports.json
```

线上建议使用：

```text
Supabase
Render PostgreSQL
Railway PostgreSQL
```

如果用 Supabase，通常需要：

```env
DATABASE_SSL=true
```

服务启动后会自动创建 `reports` 表，不需要手动建表。

## 部署后测试

部署成功后访问 Render 给你的域名，测试：

```text
1. 打开 /health，确认返回 {"ok":true,"storage":"postgres"}
2. 生成八字四柱
3. 解锁全部命理
4. 是否返回查询码
5. 查询链接是否能打开
6. 刷新后查询链接是否还能打开
```

如果已经配置 PostgreSQL，查询链接在重启后仍应可访问。
